# FUNCTIONAL REQUIREMENTS

## Backend
- Health check: `GET /health`
- Threat Actors:
  - `GET /intel/threat-actors` — list of actors
  - `GET /intel/threat-actors/analyze` — actor intelligence summary
  - `POST /intel/threat-actors/import` — add actor
- Fraud Scanning:
  - `GET /fraud/scan/sim?persist=bool` — scan HTML pages
  - `POST /fraud/ingest` — ingest single HTML
  - `GET /fraud/findings?limit=100`
- Prediction:
  - `GET /predict/attack?region=XX` — ML/AI model returns:
    `{ risk_score, likely_attack_type, likely_target, severity_level, recommended_defenses }`

### AI/ML
- Train fraud prediction model using historical findings + simulated data
- Train threat actor profiling model using mock actor dataset
- Endpoint `/predict/attack` calls ML model for real-time scoring
- Persist trained models in `ml_model.py` and `models/` folder
- Include evaluation metrics (accuracy, precision, recall)
